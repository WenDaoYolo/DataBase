

什么是数据库?
```
数据存储是开发中的核心，而数据库就是存储和管理数据的一种程序软件，例如文件接口写的数据库，但我们写的数据库存在许多漏洞
而第三方平台的数据库更成熟、强大，可直接用于工业级的生产，所以无需从零开始实现数据库，只作为学习即造轮子与用轮子的区别

目前主流且免费的第三方平台数据库软件如：MySQL(关系型数据库)、PostgreSQL(关系型数据库，同时也支持非关系型的数据组织)
```

数据库的使用方式
```
GUI图形化界面：通过鼠标点击的方式来使用数据库，降低了使用门槛，方便人查看和使用
CLI命令行界面：通过终端命令行的方式来使用数据库，效率高例如批处理、能和程序交互
程序与数据库的交互：程序连接数据库后只能通过提供的命令行操作数据库，鼠标完成无法
```

SQL语句
```
数据库会提供的一系列命令行语句用于操作数据库，像这样的语句就称为SQL语句(Structured Query Language，结构化查询语句)
```
---

PostgreSQL
```
PostgreSQL提供的图形化界面：pgAdmin 4
PostgreSQL提供的命令行界面：SQLShell、终端控制台、客户端插件
本次学习PostgreSQL下的命令行操作，Linux和Windows使用方式相同
```

初始环境配置
```
初次使用数据库需要配置相关的参数

用户名：可省略，默认为postgres
密码：必须设置，用于数据库的登录
端口号：设置数据库使用的连接端口
```

启动与连接
```
1.通过SQLShell连接
2.通过终端控制台连接
3.通过客户端插件连接
```

path环境变量
```
环境变量即定义的系统可识别的一个指针变量，方便快速寻址，在运行软件时需要定位到可执行文件所在的路径然后才能启动
例如：D:/tmp/xxx，可以将文件路径设为一个环境变量，系统就会自动识别，此时任意路径都以可运行该环境下的某个文件

1.数据库的环境变量：设置后无需手动定位路径，系统会识别
配置环境变量后重启终端，终端就能在任意路径下连接数据库

2.密码的环境变量：PGPASSWORD代表密码，设置后不再输入
临时环境变量：命令行set PGPASSWORD WdXdRr&424764
永久环境变量：新建变量，PGPASSWORD，WdXdRr&424764
```
